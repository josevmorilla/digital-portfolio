## Design Context

### Users
Primary audience is recruiters and hiring managers evaluating fit for technical roles. They need to quickly assess technical breadth, project quality, communication clarity, and professional maturity, then take a clear next action (contact, review resume, or open key project links).

Secondary audiences include peers and potential collaborators, but design decisions should optimize first-scan clarity for hiring decisions.

### Brand Personality
Voice and tone should feel efficient, analytical, and resourceful.

Emotional goals:
- Build trust and confidence quickly
- Signal technical competence without visual noise
- Keep communication concise, factual, and easy to scan

### Aesthetic Direction
Direction: minimal and professional.

Current implementation signals to preserve:
- Primarily light experience with controlled contrast
- Monochrome-neutral foundation with a muted red accent
- Clean typography (Inter) and disciplined spacing
- Subtle motion only when it supports clarity

Theme strategy:
- Keep primarily light mode as the default experience

Color guidance:
- Continue using the current muted red accent as the primary highlight
- Avoid introducing loud/saturated accent systems unless needed for meaning

### Design Principles
1. Fast credibility over decoration: prioritize scanability, evidence of work, and clear hierarchy.
2. Functional minimalism: remove non-essential visuals; every element should support comprehension or action.
3. Consistent systems: enforce repeatable spacing, typography, and component behavior across sections.
4. Accessibility-first baseline: meet WCAG 2.1 AA with strong contrast, keyboard clarity, and semantic structure.
5. Action clarity: make primary actions (contact, resume, project links) obvious and low-friction.
